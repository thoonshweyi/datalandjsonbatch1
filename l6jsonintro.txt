=>JSON (Javascript Object Notation)

=JSON
JavaScript Object Notation
Base on Text Format
Light-weighted
Does not support comments and namespaces
NoSQL database like MongoDB use JSON formats

=>JSON
Need to enclose name of key within double quotes (should not use single quotes)
But can use single quotes to enclose key names

=>JSON vs XML
JSON is more consise/lightweight/easytounderstand
XML is more text longer and look more complex

=JSON Syntax rules

JSON data to written inside curly braces
Data is presented as key value pairs
Keys should be enclosed in double quotes
Key and value should be separated by colon (:)
Comma (,) is used to separate data
